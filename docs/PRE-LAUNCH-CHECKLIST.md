## 🔴 Обязательно перед запуском

### 1. Переменные окружения (production)

В Vercel (или где хостится сайт) должны быть заданы:

- `NEXT_PUBLIC_SITE_URL` — полный URL сайта (например `https://whybao.ru`).
- `TELEGRAM_BOT_TOKEN` — токен бота для приёма заказов.
- `TELEGRAM_ORDER_CHAT_ID` — ID чата/группы, куда слать заказы.

Без них заказы не будут уходить в Telegram (API вернёт 503). Имеет смысл завести `.env.example` в репозитории (без секретов) с комментариями, чтобы не забыть при деплое.

### 2. Убрать или закрыть тестовую страницу

- Есть маршрут **`/test`** (`app/test/page.tsx`) — «сервер работает». Для продакшена его лучше удалить или закрыть в `robots` и не давать в навигации, чтобы не светить лишнюю точку входа.

### 3. Страница «Аккаунт»

- Сейчас заглушка: «Раздел в разработке». В навбаре есть ссылка на `/account`. Варианты: либо убрать пункт из меню до появления ЛК, либо оставить с явной подписью «Скоро».

## 🟠 Важно (рекомендуется)

### 4. Юридические страницы

Для официального сайта с заказами и персональными данными (ФИО, телефон, адрес) обычно нужны:

- **Политика конфиденциальности** — какие данные собираем, как храним, кому передаём (Telegram, доставка), права пользователя. Обязательно, если обрабатываете персональные данные.
- **Пользовательское соглашение / оферта** — условия использования сайта и оформления заказов (можно частично дублировать то, что уже в «Контакты»: доставка, возвраты, оплата).

Имеет смысл добавить страницы `/privacy` и `/terms` (или одну объединённую), ссылки в футер и/или в форму чекаута («Оформляя заказ, вы соглашаетесь с …»).

### 5. OG-изображения товаров

- В `app/products/[slug]/layout.tsx` для Open Graph передаётся `product.images[0]` — путь вида `/products/.../1.webp`. За счёт `metadataBase` в корневом layout Next.js должен отдавать полный URL. Стоит проверить в [Facebook Debugger](https://developers.facebook.com/tools/debug/) или [Telegram Preview](https://t.me/webpagebot), что превью товара открывается с картинкой.

### 6. Favicon и иконки

- В metadata указаны `icon`, `shortcut`, `apple` на `/logo.webp`. Для Apple и части устройств удобно иметь отдельные размеры (например 180×180 для apple-touch-icon). Не блокер, но улучшает отображение в закладках и на главном экране.

### 7. Роботы: не индексировать служебные страницы

- В `robots.ts` уже есть `disallow: ["/api/"]`. Имеет смысл добавить в disallow:
  - `/checkout` и `/checkout/success` (не нужны в поиске),
  - при желании — `/cart`, `/account`, `/test` (если страница останется).

### 8. Согласие на аналитику (Cookie / Метрика)

- Яндекс.Метрика по умолчанию использует cookies. Если хотите быть формально «по закону», можно добавить баннер/модалку «Мы используем cookie и аналитику…» с кнопкой «Принять» и не инициализировать Метрику до согласия (или показывать баннер только при первом визите и сохранять выбор в localStorage).

## 🟡 По желанию (улучшения)

### 9. Страница «Аккаунт» и заказы

- Долгосрочно: личный кабинет с историей заказов (потребуется бэкенд/БД или хранение по Telegram ID). Пока заглушка — ок.

### 10. Rate limiting / защита API

- `POST /api/orders` открыт. Для защиты от спама и злоупотреблений можно добавить лимит запросов по IP (например через Vercel Edge Config, Upstash Redis или middleware с счётчиком). Не обязательно в первый день, но перед большим трафиком — желательно.

### 11. Валидация телефона на клиенте

- В API телефон проверяется (длина цифр 10–11). На форме чекаута можно добавить маску ввода и подсказку формата (например «+7 (999) 123-45-67»), чтобы меньше ошибок и отказов.

### 12. Доступность (a11y) — как сделать

- **Клавиатура:** пройти по сайту только Tab/Enter/Space — все кнопки и ссылки должны быть достижимы, видно фокус (обводка). В Tailwind уже есть `focus-visible:ring` на кнопках — проверьте, что он не отключён глобально.
- **Скринридер:** включите VoiceOver (Mac) или NVDA (Windows), откройте главную → каталог → товар → корзина. У картинок должен быть осмысленный `alt` (у товаров — название), у кнопок без текста — `aria-label` (у вас уже есть на иконках «Поиск», «Избранное», «Меню»).
- **Модалки:** при открытии чекаут-подтверждения или корзины фокус должен уходить внутрь модалки и не «убегать» на контент сзади. Закрытие по Esc и клику по оверлею — плюс.
- **Контраст:** текст и кнопки должны быть читаемы на фоне (проверка в Lighthouse Accessibility или вручную). Primary-цвет у вас мягкий — убедитесь, что на белом/тёмном он достаточно контрастный.
- **Быстрый выигрыш:** добавить в начало `<main>` ссылку «Перейти к контенту» с `className="sr-only focus:not-sr-only"` (скрыта визуально, видна при фокусе с клавиатуры) для пользователей скринридеров.

### 13. Производительность

- Изображения товаров уже в WebP, hero тоже. При росте каталога можно включить `next/image` для карточек товаров (lazy load, srcset). Проверка в Lighthouse (Performance, LCP) — полезный финальный шаг.
